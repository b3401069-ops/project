# Agent 配置指南

## 语言偏好
- 所有回應必須使用**繁體中文**
- 程式碼註解、變數命名、提交訊息等也應盡量使用繁體中文
- 當需要與使用者溝通時，必須使用繁體中文
- **思考過程**（如使用 `think` 工具）應使用**英文**，以便於記錄和分析
- **最終回答**必須使用**繁體中文**

## Git 與 GitHub 設定
- 設定 Git 全域使用者名稱和郵箱
- 使用 GitHub CLI (`gh`) 進行認證
- 建立新的分支進行開發，避免直接在 main/master 分支上提交
- 提交訊息應使用繁體中文，並遵循規範格式（如：feat:, fix:, docs:）

## 開發流程
1. 建立功能分支：`git checkout -b feature/功能名稱`
2. 定期從主分支拉取更新：`git pull origin main`
3. 提交變更：`git commit -m "描述性訊息"`
4. 推送到遠端：`git push -u origin feature/功能名稱`
5. 建立 Pull Request：`gh pr create`

## 程式碼風格
- 程式碼應清晰、簡潔
- 添加適當的註解（繁體中文）
- 遵循專案的既有風格

## 安全性
- 不要將敏感資訊（如 API 金鑰）提交到版本控制
- 使用環境變數管理敏感配置

## 其他注意事項
- 在執行任何 Git 操作前，先確認當前狀態
- 定期備份重要變更
- 與團隊成員保持溝通
